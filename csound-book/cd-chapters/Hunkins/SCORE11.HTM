SCORE-11 Score Preprocessor for Csound: comments by Arthur B. Hunkins (1998)


Score-11 is a score preprocessor written by Aleck Brinkman originally for Music 11, at the Eastman School of Music in 1980. It was subsequently updated, through 1984. The code was then ported in the late '80's to DOS (by Craig Harris). Score-11 also functions remarkably well as a score preprocessor for the most recent 486 DOS version of Csound, for example. I have used it (in conjunction with Csound) primarily to randomize start time and duration parameters (also frequencies), with the goal of creating multiple/variable realizations of my non-realtime digital works.

The version of Score-11 that I used (DOS--dating from at least the early '90's), required the following command line:

score11 < score_input_file         (note that the < is required)

The program writes its output to the file s.out--a file suitable as (.sco) input to Csound.

Peculiarities/bugs I have observed in the current (July 1998) DOS version of Score-11 (these issues are to be addressed in the upcoming revision):

1) The "non-repeating sets" feature does not seem to work.

2) Rdeviation is undependable in that its range limits are not always observed. Also, the "deviation" is added to start time (p2) rather than duration (p3), contrary to the documentation. (The "range bug" is most unfortunate, as rdeviation could be a very useful tool for randomizing durations or start times.) Finally (and as stated), rd must be placed within an instrument block; it applies only to that block, and thus is not available globally.

3) Due to differences between Music11 (the program's original "parent") and Csound, the Advance, Purge, and Comment statements discussed in the documentation are not implemented. Also, the End statement does not work correctly in Score-11 (substitute *e instead), though this does not seem to matter much (it can be omitted).

One particular limitation of Score-11 has proven especially difficult for me: its significantly restricted random number capability. (This function has always been the primary rationale for Score-11 in my work.) Though Score-11 allows for different values of Rseed, generating an unlimited quantity of random number series, there is no way of "massaging" these numbers other than setting their limits and specifying whether or not they are integer. For example, it is not possible to rule out immediate repetitions (crucial when working with integers), to specify some minimum difference between successive values, or to make a later value dependent on a previous one. It seems that a more elaborate and powerful "musical text processor" for Csound is required to furnish such increased functionality. (Or perhaps these are features to be hoped for in upcoming revisions.)
